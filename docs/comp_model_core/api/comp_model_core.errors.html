comp_model_core.errors

Custom exceptions used throughout comp_model_core.

The library distinguishes between:

  • Compatibility issues (e.g., a model cannot be applied to a given task spec).

  • Parameter validation issues (e.g., invalid types or out-of-bounds values).

Exceptions

CompatibilityError

Raised when components are incompatible.

ParameterValidationError

Raised when a parameter value fails validation.
exception comp_model_core.errors.CompatibilityError[source]

Bases: ValueError

Raised when components are incompatible.

Examples include:

  • Applying a model to a TaskSpec it does not support.

  • Using a social model with an asocial task (or vice versa).

This exception subclasses ValueError so it integrates naturally with argument validation patterns.

exception comp_model_core.errors.ParameterValidationError[source]

Bases: ValueError

Raised when a parameter value fails validation.

Typical causes:

  • Non-numeric values where floats are required.

  • Non-finite values (NaN, inf).

  • Values outside declared bounds when bounds checking is requested.
