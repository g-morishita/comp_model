comp_model_core.events.types

Event log schema.

This module defines the event types and data structures for representing an event-log “experience trace” that can be replayed by different estimators (e.g., MLE vs Stan) without re-implementing timing logic.

The key idea is that the ordering of events is the source of truth for when social observation, choice, and outcome updates occur.

Notes

Generators typically store serialized event logs inside comp_model_core.data.types.Block.metadata under the key EVENT_LOG_KEY.

Bases: IntEnum

Discrete event types for an event stream.

The integer codes are part of the event-log contract. If you change them, you must update any downstream code that relies on the codes (e.g., Stan programs).

BLOCK_START

Marks the beginning of a block. Estimators should reset latent state here.
