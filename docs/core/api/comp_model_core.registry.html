comp_model_core.registry

Simple registries for named components.

Registries are used to decouple configuration (e.g., JSON/YAML plans) from concrete Python classes. For example, a BlockPlan can store bandit_type="bernoulli_bandit" and the caller can look up the corresponding constructor in a registry.

This module intentionally keeps the registry logic minimal.

class comp_model_core.registry.NamedRegistry[source]

Bases: Generic[T]

A mapping from string names to items.

The registry enforces unique names and provides a small convenience API.

Notes

This is not intended to be a plugin system; it is simply a structured dict with a clearer error message and a stable names() method for introspection.
