comp_model_core.interfaces.model

Computational model interfaces.

A computational model defines how an agent chooses actions and updates latent state given observations from a task environment.

This module defines:

  • ComputationalModel for asocial tasks.

  • SocialComputationalModel for tasks with social observations.

class comp_model_core.interfaces.model.ComputationalModel[source]

Bases: ABC

Abstract base class for computational models.

Subclasses should implement: - param_schema - supports() - reset_block() - action_probs() - update()

Notes

Models are stateful: they hold latent variables (e.g., Q-values). Generators and estimators should reset state at each block boundary via reset_block().
